一、准备工作

1.注册一个Github/Gitee账号

Github官网

Gitee官网

Github是国外的开源仓库，支持Web Page服务，可以为用户代理网站；Gitee是国内的开原仓库，也包含Gitee Page服务。两者没有很大的区别，但对于国内用户来说，Gitee的上传下载速度更快些。

2.创建一个测试仓库

创建测试仓库的目的是为了测试本地Git与远程仓库的ssh数据传输，因此只需要在其中一个仓库创建即可，这里笔者是在Github上创建的仓库。

在主页里面点击Repositories，然后点击New创建测试仓库。

在Repository name里面填text,点击Create创建一个测试仓库。

3.创建master分支

如果你用的github，你还需要自己创建一个master分支，gitee会自动创建master分支。进入你创建的仓库中，点击main进去创建一个名为master的分支。

接下来，我们需要将id_rsa.pub文件的内容复制到远程仓库中（这里以Github为例，Gitee同理）。首先打开自己的Github主页，点击右上角用户头像，进入Settings中，

接着在左边选项卡里面选择SSH and GPG keys选项，点击右边New SSH key添加公钥，

二、博客搭建

1.为你的个人博客创建一个远程仓库

(1)Github远程仓库

仓库的Repository name里面填你的用户名.github.io点击Create创建一个仓库。

(2)Gitee远程仓库

在右上键点创建仓库

在里面填你的个人空间地址，其他的不管，点击Create创建一个仓库。

获取个人空间地址的方法：

右上角点击设置

在个人资料->个人空间地址可以查看，方框里面的就是个人空间地址。

2.设置master分支(仅Github)

进入你创建的仓库中，点击main进去创建一个名为master的分支，然后在菜单栏选择Setting，在GitHub Pages->Source里面选择创建好的分支master，点击Save保存。之后就暂时不同Github了。

3.创建本地仓库并安装Hexo

Hexo是一种个人博客的搭框架的软件，方便我们假设个人博客。首先，在本地新建一个文件夹，用来存放博客信息。然后在文件夹里面鼠标右键打开Git Bash，输入npm命令安装Hexo:

此时在浏览器输入http://localhost:4000就可以打开新部署的网页了，如果看完了可以在git bash界面按ctrl + c 停止运行。

4.将Hexo部署到Github

首先打开我们的博客文件夹根目录，用笔记本打开配置文件_config.yml，在最底部添加

deploy:
  type: git
  repository: git@github.com:DreamStalk-ai/DreamStalk-ai.github.io.git
  branch: master

注意，这里的repository是仓库中code里面的ssh地址，不是https地址。

如果是Gitee，这里的repository就要改为git@gitee.com:minkeren/minkeren.git。

然后回到根目录中，打开Git Bash，安装Git部署插件，输入命令:

,运行完后输入以下三条命令：

hexo clean        #清除缓存文件
hexo g            #生成网站静态文件到public文件夹
hexo d            #生成网站静态文件并部署到制定Github仓库

完成后，打开浏览器，输入https://你的用户名.github.io就可以打开你的网页了。

如果是Gitee，还需要在仓库进行的服务->Gitee Page里面进行刷新，才能看得到。
